# diaryをコミットする

src/content/diaryに未コミットのものがあれば、以下の手順でコミットする：

## 実行フロー

1. **対象ファイルを確認**
   - `git status --porcelain src/content/diary src/assets` で未コミットファイルを取得

2. **新規作成か修正かを自動判別**
   - ステータスが A（新規）か M（修正）かを確認

3. **コミットメッセージを自動生成**
   - 新規の場合：`diary: YYYY-mm-dd`
   - 修正の場合：`diary: YYYY-mm-dd [変更内容の要約]`

4. **修正の場合の変更内容要約**
   - `git diff` で変更内容を確認し、簡潔な説明を自動生成
   - 例：「文言修正」「画像追加」「リンク修正」「参照を引用に変更」など

5. **対象ファイル**
   - src/content/diary
   - src/assets

## 重要な改善点
- 手動で修正内容を説明する必要がなく、git diffから自動で変更内容を分析
- 新規作成と修正を自動判別してコミットメッセージを適切に生成
- 変更内容が複数ある場合は主要な変更点を要約して表示
